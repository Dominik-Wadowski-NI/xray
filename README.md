# Resolver Map Visualization

An interactive GraphQL resolver dependency visualization that shows how your services, types, and resolvers are connected.

## Quick Start

Run web script

```
npm run web
```

## Features

### Graph Layers

The visualization presents your resolver architecture in 4 layers:

1. **Services** — Large blue/red circles representing your microservices
2. **Types** — Medium circles showing GraphQL types and their owning service
3. **Resolvers** — Small circles for each resolver field, colored by implementing service
4. **Resolver Flow Panel** — Detailed view of a resolver's implementation when clicked

### Interactions

- **Click on resolvers** (small nodes) to see their resolver flow
- **Hover** on any node to highlight it
- **Drag** to move nodes around
- **Scroll** to zoom in/out
- **Pan** by clicking and dragging the background
- **Toggle layers** using the controls on the left

### Resolver Flow Panel

When you click on a resolver node, a slide-in panel shows:

- **Summary** — Quick stats on functions, external packages, DB calls, class instantiations
- **Call Chain** — Ordered list of internal function calls
- **External Packages** — NPM packages and methods used
- **Database Calls** — All database operations
- **API Requests** — External API calls
- **Class Instantiations** — Classes created in the resolver

## Color Scheme

- **content-element-rts** — Blue (#4a9eff)
- **product-rts** — Red (#ff6b6b)

Resolvers inherit the color of their implementing service. Cross-service dependencies are shown with dashed edges.

## Data Files

The visualization reads from:

- `resolver-map.json` — Main graph data
- `{service}/resolver-flows.{resolverName}.json` — Detailed resolver implementation flows

These files are typically generated by the xray analysis tools.

## Troubleshooting

### "Failed to load resolver map data"

**Solution:** You're opening the file as `file://` directly. Use a web server as shown in Quick Start.

### Resolver flow panel shows blank/error

1. Check that the resolver-flows JSON files exist in the service folders
2. Ensure the server is still running
3. Check browser console for specific error messages

### Graph is too large or clustered

Use the zoom controls:
- **Scroll wheel** to zoom in/out
- **Drag background** to pan
- Nodes can be dragged to rearrange

## Browser Compatibility

Works on all modern browsers (Chrome, Firefox, Safari, Edge). Requires ES6 support and D3.js v7 (loaded from CDN).

## Customization

To modify the visualization, edit `resolver-map.html`:

- **Colors**: Modify `serviceColors` object around line 450
- **Node sizes**: Adjust `nodeRadii` object
- **Force simulation**: Tune the D3 forces in `initVisualization()`
- **Layout**: Adjust initial layer positions in `initVisualization()`
